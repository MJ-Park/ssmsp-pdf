# 🚧 스마트 안전관리 체계 AI 기능 개발: 위험성평가 자동화

**프로젝트 개요**
* **목표:** 위험성평가표(PDF) 업로드 시, AI가 과거 데이터를 기반으로 현재 위험성 및 감소대책의 적절성 평가 및 제안
* **핵심 전략:** RAG (Retrieval-Augmented Generation) + Few-shot Prompting
* **사용 모델:** Gemini 1.5 Pro (Vertex AI, 추후 2.5 Pro 호환 아키텍처)
* **학습 데이터:** 2024~2025년 구조물 개량공사 데이터 217건

---

## Phase 1. 데이터 전처리 및 임베딩 (Data Preparation)
> AI가 "비슷한 상황"을 찾을 수 있도록 데이터를 벡터(숫자)화하여 저장하는 단계입니다.

- [ ] **임베딩 대상 필드 추출 및 정제**
    * *설명:* 검색의 정확도를 위해 전체 텍스트가 아닌 '매칭에 필요한 필드'만 결합하여 임베딩 소스로 사용합니다.
    * *Target:* `세부작업` + `위험분류` + `위험상황결과`
    * *Metadata:* 검색된 후 LLM에게 참고자료로 넘겨줄 나머지 필드 (`현재 위험성`, `감소대책`, `안전보건조치` 등) 분리 저장

- [ ] **벡터 변환 (Vectorizing)**
    * *모델:* Vertex AI `text-embedding-004` (한국어 및 기술 문서 성능 우수)
    * *작업:* 217건의 데이터를 벡터 값으로 변환

---

## Phase 2. 검색 엔진 구축 (Vector Search Setup)
> 사용자의 질문(작업 상황)에 대해 0.1초 안에 가장 유사한 과거 사례를 찾아오는 검색소를 구축합니다.

- [ ] **Vertex AI Vector Search 인덱스 생성**
    * Phase 1에서 생성한 벡터 데이터를 업로드하여 인덱스(Index) 생성
    * 배포할 엔드포인트(Endpoint) 생성 및 인덱스 연결

- [ ] **검색 로직(Retrieval Logic) 구현**
    * *Query:* 사용자 입력 텍스트(작업 내용)를 동일한 모델(`text-embedding-004`)로 실시간 임베딩
    * *Parameter:* **Top-k = 3** (가장 유사한 사례 3개만 추출)
    * *Threshold:* 유사도 점수가 너무 낮은 데이터는 걸러내는 임계값 설정 (예: 0.6 미만은 제외)

---

## Phase 3. 프롬프트 엔지니어링 (Prompt Engineering)
> 검색된 데이터를 근거로 Gemini가 "도로교통공사 안전 전문가"처럼 판단하게 만드는 핵심 두뇌 설계 단계입니다.

- [ ] **시스템 프롬프트(System Instruction) 초안 작성**
    * *Role:* 한국도로교통공사 스마트 안전관리 시스템 AI 안전 전문가
    * *Task:* 위험요인 분석, 위험성 등급 평가(상/중/하), 감소대책 제안

- [ ] **Context 주입 구조 설계 (Few-shot)**
    * RAG로 찾아온 Top-3 사례를 프롬프트 내에 동적으로 삽입하는 템플릿 작성
    * *입력 예시:* "다음은 우리 공사의 과거 유사 사례(Context)이다. 이를 기준점(Ground Truth)으로 삼아 평가하라."

- [ ] **평가 로직 세분화 (Conditional Logic)**
    * *Low Risk (점수 낮음):* "과도한 대책 제안 지양, 현행 안전보건조치 준수로 처리"
    * *High Risk (점수 높음):* "데이터셋의 `감소대책` 스타일을 모방하여 구체적 대책 필수 제안"

- [ ] **안전장치(Fallback) 로직 추가**
    * *내용:* 검색된 유사 사례가 없거나 연관성이 떨어질 경우, 억지로 매칭하지 말고 "KOSHA(안전보건공단) 일반 지침"을 기반으로 판단하도록 지시

---

## Phase 4. 서비스 통합 및 테스트 (Integration)
> 실제 웹앱(PDF 업로드)과 AI 로직을 연결하는 단계입니다.

- [ ] **PDF 텍스트 추출 파이프라인 구축**
    * *Tool:* Google Cloud Document AI (OCR) 활용
    * 업로드된 PDF에서 '작업 내용', '위험 요인' 텍스트 블록 추출

- [ ] **LLM 요청/응답 파이프라인 연결**
    * `[OCR 텍스트] -> [Vector Search] -> [Prompt 조합] -> [Gemini 호출] -> [JSON 파싱]` 흐름 구현

- [ ] **결과 검증 (Test)**
    * *Test Case 1:* 기존 데이터에 있는 작업(예: 비계 해체) 입력 시 -> 동일/유사한 평가 나오는지 확인
    * *Test Case 2:* 데이터에 없는 작업 입력 시 -> Fallback 로직 작동 여부 확인
